# Negotex Identifier Conventions

Reference for all identifier types used across the protocol.
Kit implementors must follow these conventions exactly —
identifiers appear in topic names, database columns, manifests,
and audit records.

## Identifier types

| Identifier | Type | Example | Notes |
|---|---|---|---|
| `envelopeId` | UUID v4 | `a3f2c1d4-...` | Generated by the runtime per envelope |
| `processInstanceId` | UUID v4 | `b7e9a2f1-...` | Generated at the Trigger node |
| `processDefinitionId` | kebab-case TEXT | `loan-application` | Human-readable; appears in topic names |
| `processVersion` | semver TEXT | `1.3.0` | Structural version only — see ADR-023 |
| `nodeId` | kebab-case TEXT | `credit-check` | Unique within a process definition |
| `handlerVersion` | semver TEXT | `2.1.0` | Handler logic version — tracked in Runtime Manifest and hash chain |
| `attrKey` | camelCase TEXT | `application` | Payload attribute key; matches `input`/`output` in process definition YAML |
| `edgeTopic` | structured TEXT | `loan-application-v1.3.0.edge.validate-to-credit-check` | See topic-naming.md |
| `dlqTopic` | structured TEXT | `loan-application-v1.3.0.dlq.credit-check` | See topic-naming.md |
| `anchorReferenceId` | opaque TEXT | `contract-2024-LN-00472` | External system reference; format determined by external system |
| `policyId` | kebab-case TEXT | `banking-de-standard` | Retention policy ruleset identifier |
| `policyVersion` | semver TEXT | `2.1.0` | Version of the retention policy ruleset |

## Notes

**`processDefinitionId` and `nodeId`** must be kebab-case and must not contain dots
(dots are reserved as separators in topic names).

**`handlerVersion`** is distinct from `processVersion`. A handler logic fix bumps
`handlerVersion` only — no new topics, no drain-and-switch. A structural change
(new node, rewired edge) bumps `processVersion`. See ADR-023.

**`anchorReferenceId`** is an opaque external reference whose format is determined
by the external system. Examples by anchor type:

| `anchor_flags` value | Example `anchorReferenceId` |
|---|---|
| `CONTRACT_END` | `contract-2024-LN-00472` |
| `RELATIONSHIP_END` | `customer-DE-8821934` |
| `FISCAL_YEAR_END` | `FY2024` |
| `TRANSACTION_DATE` | `txn-2024-03-15-882341` |

**`policyId`** identifies a retention policy ruleset managed by the
`RetentionPolicyProvider` SPI. Convention: `{industry}-{jurisdiction}-{variant}`.
Examples: `banking-de-standard`, `insurance-uk-aml`, `healthcare-eu-gdpr`.
